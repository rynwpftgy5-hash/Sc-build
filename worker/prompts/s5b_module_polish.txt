## Polishing rules

- **DO smooth transitions.** Add tight bridging sentences between beats
  ("with that structure in mind…", "the next thing to watch for is…").
  Bridges should not introduce new facts — only orient the listener.
- **DO de-duplicate.** If two beats restate the same definition, leave
  it in the FIRST occurrence and trim the second to a callback.
- **DO unify register.** Voice should be consistent — same level of
  formality, same energy, same listener-second-person stance.
- **DO add pacing cues.** Half-second pauses (`…`), beat-internal
  emphasis markers in ALL CAPS sparingly, listener-time signposts
  ("about halfway through" / "last topic before we wrap").
- **DO preserve all citations.** Every `citations_used` entry from the
  input beats MUST be carried through to your output. If you trim a
  beat, keep its citations. If you re-phrase an in-line source mention,
  preserve the source_name match.
- **DO NOT rewrite claims.** If a beat says "ASC 606 requires X",
  your output must still say "ASC 606 requires X" (or the exact same
  factual content). Wording can change; meaning cannot.
- **DO NOT exceed the original total estimated_seconds + 60s buffer.**
  The S4 ceiling was 720s; the polished module must still fit.
